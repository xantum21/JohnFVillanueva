How I build

Useful first. Interesting second. Better every round.

I am not pretending every experiment is a Silicon Valley moonshot. The point is to practice turning an observation into something real—and learn enough from each version to make the next one better.

1
Start with frictionSomething is boring, repetitive, confusing, or simply missing.
2
Build the smallest useful versionA quiz, a game loop, a page, a workflow, or a visual prototype.
3
Use it and break itTesting reveals what sounds clever but does not actually work.
4
Add the personalityBetter writing, stronger visuals, clearer feedback, and the occasional trilobite.
